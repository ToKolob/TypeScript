What is TypeScript?

TypeScript is a superset of JavaScript developed by Microsoft, adding optional static typing to the language.

Why Use TypeScript?

  1. Static Typing: Helps catch errors early and improves code quality.
  2. Maintainability: Provides better documentation and code understanding.
  3. Tooling Support: Integrates well with popular IDEs and offers enhanced features.
  4. ECMAScript Compatibility: Aligns with JavaScript standards, ensuring compatibility.
  5. Community Support: Widely adopted with strong library typings.
  6. Gradual Adoption: Can be introduced gradually into existing projects.

Primitive Types

  • Are the most basic types in TypeScript
  • Equivalent to types obtained with typeof in JavaScript
  • The most commonly used ones are:
    • boolean
      • Values true or false
      • Example: let example: boolean = true
    • number
      • Does not differentiate between integers and floating points
      • Example: let example: number = 10
    • string
      • Text values
      • Example: let example: string = "hello world"

Array

  • Represents lists of data
  • Ideally with elements of the same type
  • Syntax:
    • number[]
      • Array of numbers
    • Array<number>
      • Another syntax for array

These are the most basic primitive types in TypeScript, which will be used to complement custom types or when we do not need more complex types.

Concept

  • Inference means that TypeScript can automatically detect the types of variables and values without needing to declare them explicitly
  • This brings advantages such as:
    • Autocompleting code
    • Detecting errors during development
    • Having more control over the data

How It Works

  • When we explicitly declare values, TypeScript understands their types
    • Example: const name = "John" -> String
  • It also infers types of functions and objects
    • Example: function with parameters and return
    • Example: object with defined properties

Benefits

  • We don't need to manually type everything
  • We gain productivity and code quality
  • The editor automatically shows types
  • It works even in .js files

Observations

  • TypeScript doesn't allow dynamically changing types afterward
  • JavaScript still allows dynamic and flexible types
  • But the editor can infer and show types even in JS

How to Install and Use TypeScript

Installing and using TypeScript is straightforward. Here's a step-by-step guide:

Installation

npm init

npm i --save-dev typescript

  1. Make sure you have Node.js installed on your system.
  2. Open your terminal or command prompt.
  3. Install TypeScript globally using npm (Node Package Manager): 
    npm install -g typescript
  4. Once installed, you can verify the installation by checking the TypeScript version: 
    tsc -v

Usage

  1. Create a new TypeScript file with a .ts extension, for example, app.ts.
  2. Write your TypeScript code in this file. TypeScript syntax is similar to JavaScript, with additional type annotations.
  3. To compile your TypeScript code into JavaScript, run the TypeScript compiler (tsc) followed by the filename: 
    tsc app.ts
  4. This will generate a JavaScript file with the same name (e.g., app.js).
  5. You can now run the JavaScript file using Node.js or in the browser as usual.

Configuration

Optionally, you can create a tsconfig.json file in your project directory to configure TypeScript settings. This file allows you to specify compiler options, such as target ECMAScript version, module system, output directory, and more. Here's an example tsconfig.json file:

{
  "compilerOptions": {
    "target": "es5",
    "module": "commonjs",
    "outDir": "dist"
  },
  "include": ["src/**/*.ts"],
  "exclude": ["node_modules"]
}

This configuration specifies that TypeScript should target ECMAScript 5, use the CommonJS module system, and output compiled files to the dist directory. It also includes TypeScript files from the src directory while excluding files from the node_modules directory.

By following these steps, you can easily install and start using TypeScript in your projects.

Compiling TypeScript

  • Open your terminal or command prompt.
  • Navigate to the directory where your TypeScript file is located.
  • Run the following command to compile the TypeScript file into JavaScript: 
    tsc filename.ts
    Replace filename.ts with the name of your TypeScript file.
  • After running the command, you should see a new JavaScript file in the same directory with the same name but with a .js extension, for example, app.js.

Now you can run the generated JavaScript file using Node.js or include it in your HTML files as you would with any other JavaScript file.
